mysql压缩包安装方法

2021年04月08日 Guestbook

#mysql压缩包安装方法

1）官网下载地址需要注册

https://dev.mysql.com/downloads/mysql/5.7.html#downloads

解压到自定义文件夹

2）找到mysql中的my.ini文件，将basedir后面的路径修改为你自己的安装路径，将datadir后的路径改为安装路径后data的路径（如果没有data文件就自己新建一个）

data文件内容如下

3）配置环境变量

控制面板->系统->高级系统设置->环境变量->系统环境变量中新建MYSQL_HOME

MYSQL_HOME=C:\Work_space\mysql(mysql安装目录)

Path点开，新建，%MYSQL_HOME%\bin

4）以管理员身份打开cmd命令行工具

不知道的参考： https://jingyan.baidu.com/article/c33e3f48fa1714ea15cbb5e8.html

5）输入以下指令初始化：

mysqld --initialize --console

记住红线勾的root后面的密码

6）输入安装命令

mysqld install

7）输入启动命令

net start mysql

8)登陆mysql并修改密码

mysql -u root -p

6-8如下图

9）如果出现上面的弹窗表示MySQL服务已经正常启动，使用下述命令可以为’root’@’localhost‘修改连接MySQL服务的密码

ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '123456789';（密码：123456789）

如果以后想要使用mysql，以管理员身份打开cmd命令工具，输入mysql -u root -p，回车后再输入密码即可。

mysql不同系统下的开启和关闭指令https://www.cnblogs.com/youpeng/p/10667127.html

vscode如何连接到gitee（gitlab）仓库

2021年03月30日 Guestbook

本地仓库（文件夹）连接

1）本地新建一个仓库（文件夹），可以先不进行第三步初始化直接克隆

git clone  https://gitee.com/meteorlxin/security.git(复制仓库http地址如下)

2）gitlab,gitee新建仓库

3）初始化本地仓库，克隆gitee（gitlab）仓库

git init
git remote add origin https://gitee.com/meteorlxin/security.git(复制仓库http地址如下)

4)配置用户名和邮箱，告诉仓库你是谁，私人仓库会要求填写账号密码

git config --global user.name "你的名字或昵称"
git config --global user.email "你的邮箱"

5)进入克隆仓库目录

git pull origin master

6)推送修改的文件

git add . #将当前目录所有文件添加到git暂存区
git commit -m "first attemp" #提交并备注提交信息
git push origin master #将本地提交推送到远程仓库

7）谨慎:可能远程仓库中文件和本地文件冲突（readme文件），想要强行推送

git push origin master -f

linux如何使用GDB调试C/C++程序

2021年03月26日 Guestbook

linux如何使用GDB调试C/C++程序

1）安装gdb

rpm -qa | grep gdb（查看是否有gdb）
apt-get update（如果无root权限，需要用sudo apt-get update）
apt-get install gdb
gdb -v(查看是否安装好)

centos系统需要使用yum install gdb因为linxu不同系统的安装指令有区别，详细见以下链接

https://blog.csdn.net/fatshaw/article/details/5671708

2）编译gdb调试文件

gcc main.cpp -o main(一般的编译main.cpp的过程)

此编译过程生成的main文件不能使用GDB进行编译，需要末尾加入 -g,生成的文件才可以使用GDB编译

gcc main.cpp -o main -g

3）开始gdb调试

gdb ./main

具体效果如下

4）gdb调试的常用指令

调试指令                              
(gdb) break xxx
(gdb) b xxx    
在源代码指定的某一行设置断点，其中 xxx 用于指定具体打断点的位置。

(gdb) run
(gdb) r    
执行被调试的程序，其会自动在第一个断点处暂停执行。（一般效果就是把main函数运行完）

（gdb）start
（gdb）s 
与gdb run 相似（一般效果就是运行到main函数第一行，就停下，可以接上gdb n 一行一行代码调试）

(gdb) continue
(gdb) c
    当程序在某一断点处停止运行后，使用该指令可以继续执行，直至遇到下一个断点或者程序结束。

(gdb) next
(gdb) n    
令程序一行代码一行代码的执行。

(gdb) print xxx
(gdb) p xxx    
打印指定变量的值，其中 xxx 指的就是某一变量名。

(gdb) list
(gdb) l    
显示源程序代码的内容，包括各行代码所在的行号。

(gdb) quit
(gdb) q
    终止调试。

5）gdb可视化调试

gdb ./main -tui 回车
再回车
gdb start 回车
gdb n   回车
再每打一次回车程序运行一行
