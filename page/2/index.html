centos使用make指令报错：No targets specified and no makefile found.Stop的解决方法

2021年03月24日 Guestbook

centos使用make指令报错：No targets specified and no makefile found.Stop的解决方法

有六种可能的解决方法

第一种：

yum update 
yum install gcc build-essential

然后执行./configure，make看问题是否解决

第二种：

apt-get update
apt-get install gcc build-essential

然后执行./configure，make看问题是否解决

第三种：

执行./configure指令后查看解压的文件里是否生成makefile文件。

ls -a

里面可能有Makefile.am和Makefile.in但是没有Makefile文件

这个可能是因为geany下载版本太旧了，找到最新版本链接下载

第四种：

如果没有安装其他依赖先安装依赖

yum install gcc gcc-c++ autoconf automake
yum -y install zlib zlib-devel openssl openssl-devel pcre pcre-devel （安装依赖zlib、openssl和pcre）

第四种：

可能没有安装gcc   

yum -y install gcc

使用下面指令如果有gcc版本号则安装成功

gcc -v

第五种：

主要就是不直接使用./configure命令，而是使用./configure -prefix=/usr/local -with-shared -without-debug命令

wget http://ftp.gnu.org/pub/gnu/ncurses/ncurses-5.6.tar.gz
tar zxvf ncurses-5.6.tar.gz
 ./configure -prefix=/usr/local -with-shared -without-debug
make
make install

访问github页面太慢的解决方法

2021年03月22日 Guestbook

访问github页面太慢的解决方法

1.（免费方法，不经常使用github上传代码）添加GitHub.com的ip到host文本中的

2.（付费方法，经常使用github）谷歌浏览器中安装付费插件，进行翻墙。

1.添加GitHub.com的ip到host文本中的

1）使用ip搜索网站搜索github.com的IP（不同电脑搜索的ip有差异）

https://www.ipaddress.com/ ip查询网站


2）找到 此电脑->WindowsC:->Windows->system32->Drivers->etc->host (路径不同电脑可能有点不同，可以搜索system32来找到host文件)

3）host文件默认无法修改，鼠标右击host->属性->安全

依次点击自己电脑用户名->编辑

依次点击自己电脑用户名->完全控制->确定

依次全部确定后用记事本或者natepad++打开host文件，将查询到的github.com的IP按如图格式复制到文本末（可以使用多个不同ip）

4）win+r打开cmd命令，输入ipconfig /flushdns,回车刷新dns，

输入ping github.com，回车查看ip连接是否通畅。

5）现在就能打开github了，如果还觉得慢了

http://tool.chinaz.com/dns?type=1&host=www.github.com&ip= ip查询网站

找到TTL值最小的ip复制，参考2-4步骤

2.谷歌浏览器中安装付费插件，进行翻墙。

https://meteorlxin.github.io/MeteorLXIN.github.io/2021/03/25/Chrome%E8%B0%B7%E6%AD%8C%E6%B5%8F%E8%A7%88%E5%99%A8%E5%AE%89%E8%A3%85Ghelper/
